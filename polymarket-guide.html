CHAPTER 01

Polymarket 基础认知

理解预测市场的核心机制，掌握 Polymarket 的基本运作原理

🎲

什么是预测市场

Polymarket 是一个去中心化预测市场平台，用户可以针对现实世界事件（政治、加密、体育、经济等）的结果进行交易。每个市场都是一个二元问题（Yes/No），通过价格反映事件发生的概率。

去中心化 · Polygon 链
📊

价格 = 概率

价格范围固定在 $0.00 – $1.00 之间。$0.65 的 Yes 价格意味着市场认为该事件有 65% 的概率发生。正确预测每股结算为 $1，错误预测结算为 $0。

概率定价机制
💎

USDC 与 Polygon

所有交易使用 Polygon 网络上的 USDC 稳定币结算。用户需要 Polygon 钱包（如 MetaMask）并持有 MATIC 作为 Gas 费。入金支持信用卡、加密转账和跨链桥。

USDC · MATIC Gas
🔮

二元市场结构

每个市场都有「是」(Yes) 和「否」(No) 两种份额。两个方向的价格之和始终趋近于 $1。你可以在任何时候卖出持仓获利了结，无需等到市场结算。

提前退出机制
📈

交易示例

以 $0.35 买入 200 份 Yes → 成本 $70。若价格上涨至 $0.65 卖出 → 收入 $130，利润 $60（+85.7%）。若持有至结算且正确 → 每股 $1，总收入 $200，利润 $130。

低买高卖 · 概率价差
🌐

市场覆盖范围

涵盖政治选举、加密价格、体育赛事、经济指标、天气温度、娱乐事件等数十个类别。每天有 500+ 活跃市场，新市场不断创建。

多品类覆盖
CHAPTER 02

交易机制详解

深入了解 Polymarket 的两种交易模式、订单类型、CLOB 架构和 API 体系

🎯

预测模式（新手友好）

直观显示事件概率百分比，无需理解 K 线图或订单簿。适合刚接触预测市场的用户，操作简单直接，一键买入 Yes 或 No 份额。

推荐新手使用
📉

交易模式（专业玩家）

完整的交易界面，提供 K 线图、订单簿深度图、限价单、止损单等高级功能。适合有交易经验的用户进行精确的价格操作。

高级交易工具

🏦 CLOB 中央限价订单簿架构

Polymarket 使用 CLOB (Central Limit Order Book) 作为核心交易引擎，而非 AMM（自动做市商）。这意味着所有交易都是点对点撮合的，提供更优的价格发现和更低的滑点。

📡

Gamma API

gamma-api.polymarket.com

只读市场数据接口，提供事件元数据、市场列表、价格快照、历史成交等信息。无需认证即可访问。

只读 · 免认证
💱

CLOB API

clob.polymarket.com

交易操作接口，负责订单创建、签名、提交、取消等核心交易功能。需要 L1/L2 认证。

交易核心 · 需认证
🌉

Bridge API

bridge-api.polymarket.com

跨链存提款接口，处理 USDC 从不同链到 Polygon 的桥接操作，以及资金存取功能。

CHAPTER 03

核心交易策略

从低风险套利到高赔率投机，掌握预测市场的五大主流策略

01

🎯 概率价差交易（最核心策略）

你赚的不是事件的对错，而是市场预期的价差。核心逻辑：比市场更早发现情绪拐点，在概率低估时买入，在概率高估时卖出。

胜率：~60% 风险等级：中等 适合：所有交易者
02

📊 情绪交易法

追踪三大指标：推特话题热度、KOL 态度转变、散户风向一致性。当全网观点高度一致时，就是反转信号。不在新闻刚出的前两小时情绪亢奋期进场。

胜率：~55% 风险等级：中等 关键：信息速度优势
03

🏁 尾盘扫单套利（新手最推荐）

事件快结束时（最后 5 分钟）买入，此时趋势基本确认，概率 >97%。设置止损线：若概率跌到 0.90 即卖。实测胜率约 80%，但单笔利润较小。

胜率：~80% 风险等级：低 注意：费后利润薄
04

🔄 跨市场套利

对比 Polymarket、Kalshi、PredictIt 等不同平台上同一事件的价格差异，低价买入高价卖出锁定无风险利润。窗口期通常只有几秒到几分钟，需要自动化工具。

胜率：~90%+ 风险等级：低 门槛：HFT 基础设施
05

📈 提供流动性（做市商策略）

在订单簿两侧同时挂买单和卖单，赚取买卖价差。2026 年新规后 Maker 端 0 手续费 + 返佣，是当前最受鼓励的策略方向。需要较大资金量和做市算法。

胜率：取决于算法 风险等级：中等 门槛：资金 + 技术

CHAPTER 06

2026 年规则巨变

2026 年 2 月 18 日 Polymarket 进行了史上最大规模规则调整，彻底改变了交易机器人格局

变更一
🔓 取消 500ms Taker 延迟
订单即时执行，不再有取消窗口。旧有的"延迟套利"策略（包括著名的月入 $515K 的 99% 胜率机器人）彻底失效。这对速度型 Taker 策略是致命打击。
变更二
💰 引入动态 Taker 费率
动态 Taker 费率公式
费率 = C × 0.25 × (p × (1 − p))²
峰值费率 ≈ 1.56%（在 p = 0.50 时）
极端概率（p→0 或 p→1）费率趋近于 0
这个公式意味着：在概率最不确定的市场（50%）交易成本最高，而在几乎确定的市场交易成本极低。费率结构鼓励在确定性高的市场提供流动性。
变更三
🏟️ 费率扩展至体育市场
新的费率结构扩展到 NCAA、Serie A 等体育赛事市场。此前体育市场费率较低，现在与传统市场统一。
⚠️

战略转变：Taker 套利 → Maker 做市

2026 年后，Maker 策略（提供流动性）在结构上受鼓励：0 手续费 + 返佣奖励。所有盈利机器人必须转向 Maker 策略。旧有的 Taker 套利模式由于动态费率的存在而丧失盈利空间。

🔧 2026 年可行机器人技术要求

🔌

WebSocket 优先

放弃 REST 轮询，改用 WebSocket 实时数据流。订单簿的毫秒级更新是盈利关键。

必需
💳

费率感知签名

订单必须包含 feeRateBps 字段，精确计算并动态调整报价以覆盖 Taker 费用。

必需
⚡

<100ms 撤单/替换

做市订单的取消和替换循环必须在 100ms 内完成。更高的延迟意味着库存风险暴露。

高门槛
🛡️

Maker-Only 模式

设置 Post-Only 订单标志，确保始终作为 Maker 成交，避免触发 Taker 费用。

推荐

存活策略的共同特征

仅存的 4 个盈利策略有一个共同点：它们与手动交易者竞争小众市场，而不是与其他机器人竞争。核心洞察——"找到一个更弱的对手群体比找到一个聪明的策略更重要。"

⚠️

重要警示

生存者偏差极端严重——每一个 $75K 成功故事背后，有数百个默默亏钱的失败机器人。"25 个策略，4 个存活"才是最诚实的基准。策略一旦公开就会被复制，利润边际迅速压缩。OpenClaw 本身不保证盈利——多个 OpenClaw 机器人已被清算。私钥导入存在资金被盗风险。

CHAPTER 08

实战指南

从零开始构建和部署你的 Polymarket 交易系统

🛤️ 两条入门路径

🚪

路径一：Gate 交易所（推荐新手）

通过 Gate App 的 Polymarket 模块参与，直接用现货 USDT 交易，无需链上操作、无需钱包、无需 Gas 费。体验和常规交易所交易完全一致。

零门槛 · 5 分钟上手
🔐

路径二：直接使用 Polymarket

准备 Polygon 钱包（MetaMask）、存入 USDC、获取 MATIC 作为 Gas 费。连接钱包后可直接在 polymarket.com 交易或通过 API 编程交易。

需要钱包 · 链上操作

🤖 构建交易机器人的步骤

1

选择 SDK

Python 选 py-clob-client（官方）或新的 py-sdk；TypeScript 选 @polymarket/clob-client-v2；Rust 选 rs-clob-client（追求最低延迟）；Go 选社区维护的 polymarket-go-sdk。

2

创建 API 密钥（L2 认证）

使用钱包签名 EIP-712 消息创建 API 凭证（key + secret + passphrase）。绝对不要在主网代码中硬编码私钥！使用环境变量或 KMS 管理。

3

设计策略

明确你的优势来源：数据速度？LLM 推理能力？小众市场认知？API 先行验证数据源，确认你能获取别人没有的信息。从小众市场开始，避免与专业做市商正面竞争。

4

回测与模拟

使用历史数据回测策略。Polymarket 提供历史 Gamma API 数据。先用小额资金（$20-50）进行真实环境测试，不要在回测中过拟合。

5

部署与监控

部署到低延迟服务器（推荐 AWS us-east-1 或离 CLOB 服务器近的区域）。设置 Telegram/邮件告警、自动止损、仓位上限。持续记录所有交易日志用于复盘。

📋 风险控制清单

💰

资金管理

单笔交易不超过总资金的 20%。连错两单立即停止并复盘。绝不扛单，严格止损。设置每日亏损上限。

铁律
🔒

安全

使用专用钱包存放交易资金。私钥通过环境变量或 KMS 管理。定期检查 API 密钥权限。警惕钓鱼合约和恶意前端。

关键
📊

监控

实时监控仓位、未实现盈亏、API 速率限制。设置异常检测（异常大单、连续亏损、API 异常）。机器人崩溃时自动平仓保护。

重要
📝

复盘

每周分析盈亏来源、胜率变化、策略衰减迹象。市场环境变化时及时调整参数。不要迷恋一个正在失效的策略。

🏗️ 主流架构模式对比

🧠

Multi-Agent 协作架构

代表：TradingAgents, Swarm Trader, AI-Trader

模拟华尔街分工：分析师搜集数据 → 多空研究员辩论 → 交易员制定方案 → 风控官审批。每个角色由独立 LLM Agent 扮演，最终由交易员汇总各方意见执行决策。

最热门 · 71K+ Stars
🎓

端到端强化学习

代表：FinRL, Qlib (RD-Agent)

将交易建模为马尔可夫决策过程，使用 PPO/DDQN/SAC 等算法直接学习最优策略。微软 RD-Agent 用 LLM 自动挖掘因子、设计模型、进化策略。

微软 · 学术前沿
📊

LLM 直接推理

代表：Grok 4, Claude Portfolio, AlphaTrader

完全不依赖传统量化模型，将新闻/财报/宏观数据直接喂给 LLM，由 LLM 给出多空判断和仓位建议。Grok 4 在 8 模型竞赛中以 +7% 领先。

AI 交易的真相：少数赢，多数输

11 个公开 AI 交易竞技场中，仅 2 个竞技场的中位数模型盈利。最佳表现者（Grok 4、北大 Agentic AI 系统）展示了真实 Alpha，但大多数公开实验亏损或跑输市场。核心问题：LLM 普遍存在过度交易、时机错误、仓位管理不当。传统回测对 LLM 无效（训练数据已包含历史信息），只有实盘测试具有参考价值。

🛠️ 构建美股交易机器人的推荐路径

1

确定技术路线

入门：Backtrader + yfinance + Alpaca 纸交易 — 经典可靠，社区成熟。
进阶：Qlib + RD-Agent — 自动因子挖掘，微软背书。
前沿：TradingAgents / Swarm Trader — Multi-Agent LLM 架构，零代码实验。

2

选数据源

免费：yfinance（日线）+ SEC EDGAR（财报）+ Finnhub（新闻/情绪）。
付费：Polygon.io（实时逐笔）、Alpha Vantage、Quandl。大多数开源项目的免费数据足以支撑研究阶段。

3

回测 → 纸交易 → 小额实盘

先用 Backtrader/Zipline 回测验证策略逻辑 → Alpaca Paper Trading 模拟真实市场环境（至少 1-3 个月）→ 小额资金（$500-2000）实盘验证。警惕 LLM 的 look-ahead bias：训练数据可能已包含"未来"信息。
